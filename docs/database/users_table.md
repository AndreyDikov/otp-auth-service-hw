# 📋 users

> `users` — таблица для хранения зарегистрированных пользователей. В ней сохраняются логин, хэш пароля, роль 
> пользователя и дата создания записи

## 🧾 Поля таблицы

| Поле            | Тип            | Может быть `null` | Значение по умолчанию          | Описание                                  |
|-----------------|----------------|------------------:|--------------------------------|-------------------------------------------|
| `id`            | `bigint`       |                 ❌ | `generated always as identity` | Уникальный идентификатор пользователя     |
| `login`         | `varchar(100)` |                 ❌ | —                              | Логин пользователя                        |
| `password_hash` | `varchar(255)` |                 ❌ | —                              | Хэш пароля пользователя                   |
| `role`          | `varchar(20)`  |                 ❌ | —                              | Роль пользователя                         |
| `created_at`    | `timestamp`    |                 ❌ | `now()`                        | Дата и время создания записи              |

---

## 🔒 Ограничения таблицы

| Ограничение               | Поле / поля | Описание                                                          |
|---------------------------|-------------|-------------------------------------------------------------------|
| `pk_users`                | `id`        | Первичный ключ таблицы. Значение `id` уникально для каждой записи |
| `uq_users_login`          | `login`     | Значение `login` должно быть уникальным                           |
| `chk_users_role`          | `role`      | Разрешены только значения: `ADMIN`, `USER`                        |
| `ux_users_only_one_admin` | `role`      | Разрешает создать только одного пользователя с ролью `ADMIN`      |
