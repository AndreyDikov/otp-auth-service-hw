# 📋 otp_config

> `otp_config` — таблица для хранения текущей конфигурации OTP-кодов. В таблице может быть только одна запись, которая 
> определяет длину генерируемого кода и время его жизни

## 🧾 Поля таблицы

| Поле          | Тип         | Может быть `null` | Значение по умолчанию | Описание                           |
|---------------|-------------|------------------:|-----------------------|------------------------------------|
| `id`          | `smallint`  |                 ❌ | `1`                   | Уникальный идентификатор настройки |
| `code_length` | `int`       |                 ❌ | —                     | Количество символов в OTP-коде     |
| `ttl_seconds` | `int`       |                 ❌ | —                     | Время жизни OTP-кода в секундах    |
| `updated_at`  | `timestamp` |                 ❌ | `now()`               | Дата и время последнего обновления |

---

## 🔒 Ограничения таблицы

| Ограничение                  | Поле / поля   | Описание                                                                                     |
|------------------------------|---------------|----------------------------------------------------------------------------------------------|
| `pk_otp_config`              | `id`          | Первичный ключ таблицы. Значение `id` уникально для каждой записи                            |
| `chk_otp_config_single_row`  | `id`          | Разрешено только значение `1`. Вместе с primary key не позволяет создать больше одной записи |
| `chk_otp_config_code_length` | `code_length` | Значение должно быть в диапазоне от `4` до `10`                                              |
| `chk_otp_config_ttl_seconds` | `ttl_seconds` | Значение должно быть больше `0`                                                              |
